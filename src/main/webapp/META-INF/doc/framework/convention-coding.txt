#
# 业务分层
#
域对象（Domain）：与数据库中的记录对应，属性统一采用驼峰式命名，属性类型统一采用对象方式；
数据库字段统一采用小写字母加下划线形式；
编码时，需要注意与基础类型的转换问题，容易导致NullPointerException异常。

数据访问对象（Dao）：与业务无关的数据操作，保持原子性和通用性，Dao与Domain一一对应。

业务对象（Service）：用于业务逻辑的封装，Service可通过调用多个Dao来完成业务操作。

控制器（Controller）：一个控制器可调用多个Service来协调业务操作，禁止在Controller中直接对Dao进行操作；
同时，也应积极避免在Controller中出现组合SQL语句或查询条件的操作，避免输出视图代码。

视图（View）：用于数据展示，可以在View中进行简单的数据格式化或运算等操作，杜绝在视图中处理业务逻辑。
一个View可以被其他View包含，也可以被多个Controller调用。
